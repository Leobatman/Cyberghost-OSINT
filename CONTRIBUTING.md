# Contribuindo para o CYBERGHOST OSINT

Primeiramente, obrigado por considerar contribuir com o CYBERGHOST OSINT! É graças a pessoas como você que esta ferramenta continua evoluindo.

## Código de Conduta

Este projeto e todos os participantes estão sob o [Código de Conduta](CODE_OF_CONDUCT.md). Ao participar, você concorda em manter este código.

## Como posso contribuir?

### Reportando Bugs

Antes de reportar um bug, verifique se:
1. Você está usando a última versão
2. O bug já não foi reportado nas [issues](https://github.com/cyberghost/cyberghost-osint/issues)
3. O problema é realmente um bug e não um erro de configuração

**Como reportar:**
- Use o template de [Bug Report](.github/ISSUE_TEMPLATE/bug_report.md)
- Inclua passos claros para reproduzir
- Adicione logs e informações do sistema
- Descreva o comportamento esperado vs. real

### Sugerindo Melhorias

**Como sugerir:**
- Use o template de [Feature Request](.github/ISSUE_TEMPLATE/feature_request.md)
- Explique o problema que sua feature resolve
- Descreva como você imagina a implementação
- Inclua exemplos de uso

### Pull Requests

1. **Fork** o repositório
2. **Clone** seu fork: `git clone https://github.com/seu-usuario/cyberghost-osint.git`
3. **Crie uma branch** para sua feature: `git checkout -b feature/nome-da-feature`
4. **Faça suas alterações**
5. **Execute os testes**: `make test`
6. **Commit** suas mudanças: `git commit -m 'feat: adiciona nova funcionalidade'`
7. **Push** para a branch: `git push origin feature/nome-da-feature`
8. **Abra um Pull Request**
